Form Submission: GET vs POST

This demo lets you compare how form data is sent using GET and POST.

GET method (data appears in the URL)



After submitting, look at the browser’s address bar. You’ll see something like:
?name=Alice appended to the URL.

POST method (data hidden from the URL)



After submitting, the address bar will not change. The data is sent in the request body, invisible to the user.

Note: Since this file is opened locally, the POST request won’t actually process anywhere — but you’ll see the difference in the URL bar between GET (shows data) and POST (doesn’t show data).
